Kenya-Localized AI Medical Assistant

A comprehensive project proposal for developing AI-driven clinical decision support tailored for Kenyan healthcare professionals

Executive Summary

Form Factor

Website (Phase 1) + Android app (Phase 2)

Core Functionality

AI-driven clinical decision support, medical calculators, and curated Kenyan guideline library

Target Audience

Doctors, medical students, healthcare institutions in Kenya

Data Sources

Kenya Ministry of Health (MoH), WHO-AFRO, KEMRI, KEML, local care pathways

Objectives & Success Criteria

Primary Objectives

  • • Deliver safe, explainable AI assistant with differential diagnoses aligned with Kenyan guidelines
  • • Provide fast, reliable medical calculators and searchable reference library
  • • Ensure compliance with Kenya's Data Protection Act (DPA, 2019)
  • • Maximize usability for Kenyan clinicians, including rural practitioners

Performance Targets

  • • p95 response time < 6s for AI queries
  • • p95 response time < 3s for calculators
  • • ≥95% uptime for MVP
  • • Zero critical errors in validated test sets

Quality Metrics

  • • 100% AI outputs include citations
  • • ≥80% clinicians report actionable outputs
  • • 100% DPA compliance
  • • Zero PHI breaches in testing

Development Phases

Phase 1: Website + Backend (5 Weeks)

Deliver a secure, web-based AI Medical Assistant with core functionality for clinicians and students.

Backend APIs (FastAPI)

  • • JWT authentication with RBAC
  • • AI decision support endpoint
  • • Medical calculators (BMI, GFR, Wells, etc.)
  • • Full-text search for guidelines
  • • Subscription management

Website (Next.js)

  • • Secure login with 2FA option
  • • AI Console with clinical input forms
  • • Calculator hub with validation
  • • Reference library with search
  • • Admin portal for content management

Phase 2: Flutter Android App (5 Weeks)

Extend functionality to mobile app with offline capabilities and push notifications.

Mobile Features

  • • AI Console with quick-entry flows
  • • On-device calculator execution
  • • Cached reference library (50MB max)
  • • Offline mode for calculators
  • • Push notifications via Firebase

Mobile Enhancements

  • • Sync endpoints for bookmarks
  • • Mobile-specific telemetry
  • • Rate limiting with quotas
  • • Kiswahili UI toggle
  • • Low-storage optimization

Technical Architecture

Frontend Stack

  • • Next.js with SSR
  • • Flutter for Android
  • • WCAG 2.1 AA compliance
  • • Dark mode support

Backend Stack

  • • FastAPI with Uvicorn
  • • PostgreSQL 15 with RLS
  • • Redis for caching/queues
  • • S3-compatible storage

AI & Search

  • • RAG with Qdrant vector DB
  • • GPT-4o-mini via proxy
  • • SentenceTransformers
  • • Kenya-specific priors

AI Design & Safety

Safety Guardrails

  • • Drug-drug interaction validation
  • • Dose range checks by age/weight
  • • Red-flag triggers for urgent referral
  • • Confidence thresholds for refusal

Quality Metrics

  • • Top-3 differential recall ≥90%
  • • Treatment concordance ≥95%
  • • Hallucination rate <1%
  • • 100% citation coverage
